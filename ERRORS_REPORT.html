🏗️ Архитектурные проблемы (11 проблем)

God Objects (слишком большие классы)

CRITICAL GameInstaller.java: 389 строк
Обрабатывает: скачивание, распаковку, отслеживание прогресса, паузу/возобновление, нативы
Разделить на:
- GameDownloader (управление загрузками)
- LibraryExtractor (распаковка библиотек)
- ProgressReporter (отслеживание прогресса)
- PauseController (пауза/возобновление)
CRITICAL GameLauncher.java: 335 строк
Обрабатывает: разрешение Java, построение classpath, управление нативами, запуск процесса
Разделить на:
- JavaRuntimeSetup
- ClasspathBuilder
- NativesManager
- GameProcess

Нарушение SOLID

HIGH Single Responsibility Principle: VanillaManager.java
Одновременно: подготовка игры, управление Java, распаковка нативов
Отделить JavaRuntimeManager и NativesExtractor
HIGH Open/Closed Principle: Классы жестко связаны
UI напрямую создает GameLauncher, VanillaManager, FabricManager
Использовать Factory pattern или Dependency Injection

Неправильные паттерны

HIGH Singleton anti-pattern: ConfigManager, CacheManager
Статические getInstance() создают глобальное состояние, затрудняют тестирование
Использовать Dependency Injection вместо singletons
MEDIUM Отсутствие абстракции: ModAPI.java, ModrinthAPI.java
Две реализации API модов без общего интерфейса
Создать interface ModProvider для обоих
MEDIUM Циклические зависимости:
GameLauncher → VanillaManager → GameInstaller → VanillaManager
Использовать Dependency Injection для разрешения

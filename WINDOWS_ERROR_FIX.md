# Решение ошибки "CreateProcess error=193" - Пошаговая инструкция

## Что означает эта ошибка?

```
CreateProcess error=193, %1 не является приложением Win32
```

Windows не может запустить Java потому что:
- Путь содержит пробелы (например `java 8\bin\java`)
- На Windows установлена 32-битная Java вместо 64-битной
- Скачалась Linux версия Java на компьютер с Windows

## Быстрое решение (для большинства случаев)

### Шаг 1: Удалите установленную Java

1. Закройте лаунчер DanKertCraft
2. Откройте проводник
3. Перейдите в папку где лежит `DanKertCraft.jar`
4. **Удалите папку `runtime`** (если существует)
   - Полный путь: `C:\...\runtime\`

### Шаг 2: Переустановите Java

1. Запустите DanKertCraft
2. Нажмите на кнопку запуска игры
3. Лаунчер автоматически скачает и установит правильную Java
4. Дождитесь завершения (может занять 1-2 минуты)

## Если это не помогло - проверка и диагностика

### Проверка 1: Разрядность Windows

1. Нажмите `Win + Pause` или откройте "Параметры" → "О системе"
2. Найдите строку "Тип системы"
3. **Должно быть**: `64-разрядная ОС, процессор x64`
4. **НЕ должно быть**: `32-разрядная ОС`

Если 32-разрядная - вам нужна 32-битная Java.

### Проверка 2: Наличие Java в системе

1. Откройте Command Prompt (cmd.exe)
2. Напечатайте: `java -version`
3. Нажмите Enter

**Если выводит версию** - Java установлена в системе:
```
openjdk version "21.0.1" 2023-10-17
```

**Если выводит ошибку** - Java не установлена (это нормально, лаунчер скачает свою)

### Проверка 3: Правильность скачанной Java

1. Откройте проводник
2. Перейдите в папку лаунчера
3. Откройте `runtime\java17\bin\`
4. **Должен быть файл `java.exe`**

Если там `java` без расширения или файл отсутствует:
- Удалите папку `runtime`
- Попробуйте еще раз запустить лаунчер

### Проверка 4: Просмотр логов

Логи лаунчера содержат информацию о проблеме:

1. Найдите файл лога (обычно в папке лаунчера): `latest.log` или похожий
2. Откройте его текстовым редактором
3. Ищите строки:
   - `CreateProcess error=193` - ошибка запуска
   - `Java найдена:` - путь к используемой Java
   - `Java путь:` - подтверждение пути

## Если ничего не помогло

### Вариант 1: Явно установите Java

Если автоматическая загрузка не работает:

1. Откройте Java.com в браузере
2. Скачайте **Java 21 (64-bit)** для Windows
3. Установите Java в стандартную папку (`C:\Program Files\Java\`)
4. Перезагрузитесь

После этого лаунчер будет использовать системную Java.

### Вариант 2: Скачайте Java вручную

1. Перейдите на **adoptium.net** (рекомендуется)
2. Скачайте **Eclipse Temurin 21 LTS** для:
   - **Платформа**: Windows
   - **Архитектура**: x64
   - **Тип**: JRE (не JDK)
3. Установите в `C:\Program Files\Java\`
4. Перезагрузитесь

## Частые ошибки при исправлении

❌ **Неправильно**: Скачиваете 32-битную Java на 64-битную Windows
✅ **Правильно**: Скачиваете 64-битную Java на 64-битную Windows

❌ **Неправильно**: Удаляете только файл `java.exe`
✅ **Правильно**: Удаляете всю папку `runtime\java*`

❌ **Неправильно**: Устанавливаете Java в папку с пробелами
✅ **Правильно**: Устанавливаете в `C:\Program Files\Java\` (пробелы нормальны, ProcessBuilder их правильно обработает)

## Когда обращаться за помощью

Если ошибка остается:

1. Сделайте скриншот ошибки и логов
2. Укажите:
   - Версию Windows (10, 11, и т.д.)
   - Разрядность Windows (32 или 64 бит)
   - Путь к папке с лаунчером
   - Полный текст ошибки из окна лаунчера

## Технический справочник

### Структура папок (нормальная)

```
DanKertCraft.jar          ← Сам лаунчер
runtime/
├── java8/
│   └── bin/
│       ├── java.exe       ← Файл Java для Windows
│       ├── java.dll
│       └── ...
├── java11/
├── java17/
└── java21/
instances/
├── 1.20.1/
├── 1.19.2/
└── ...
```

### Как проверить что это Windows версия Java

Откройте Command Prompt и выполните:
```
cd C:\путь\к\runtime\java17\bin
java -version
```

Должно вывести одно из:
```
openjdk version "17.0.9" ... "Windows"   ← Правильно
java.exe                                  ← Запустится файл
```

### Если выводит ошибку

```
Error opening terminal definition file
The system cannot find the file specified
```

Это значит что скачалась неправильная версия Java - удалите `runtime` и попробуйте еще раз.
